<img> 요소에 svg를 불러올때 크기를 못 가져온다면 (IE)

2018-01-05

SVG(Scalable Vector Graphics) 사용용도가 매우 다양합니다. 직접 <svg> 태그를 이용하여 삽입하는 방법, css 에서 background-image 로 이미지를 표시하는 방법, <img> 태그를 이용해서 svg의 경로를 불러오는 방법, <object> 태그를 통해 출력하는방법이 있습니다. 오늘은 <img> 태그를 이용하여 사용할때 발생할 수 있는 상황에 대해서 알아보고자합니다.

IE에서 <img>요소의 svg는 크기를 계산 못 한다?

정확히는 문서에 출력되지 않은, DOM으로 svg를 이미지 형태로 가져온다면 크기 계산을 하지 못합니다.

IE이외의 브라우저에서는 해당 svg의 크기를 반환하지만, IE에서는 0을 반환합니다.

해결방법

정석적인 해결방법

해당 svg를 XMLHttpRequest로 요청하여, svg의 루트요소에 있는 width랑 height를 가져오는 방법이 있습니다.

SVG의 구조

SVG Attribute 파싱

svg는 XML 형태이기 때문에 XMLHttpRequest 의 responseXML로 파싱된 XML을 사용할 수 있습니다.

하지만 이 방법의 경우, 스크립트가 실행되는 동일한 서버에 있거나 CSRF가 허용된 svg만 사용할 수 있다는 단점이 있습니다.

문서에서 삽입하고, 계산 후, 꺼내오기

가장 확실하게 아무런 SVG라도 사용 할 수 있는 방법입니다.

이 방법의 경우 일반적인 크기로는 반환이 안되고, offsetWidth 와 같은 렌더링된 값을 사용해야한다는 것에 유의해주세요.
